Default path

Deterministic checks only. No third-party AI provider is used unless the optional review box is enabled for that run.

Optional AI review

Per-run opt-in. Adds an external review step after deterministic extraction and scoring.

What the evaluator checks

Skill structure, trigger guidance, examples, failure handling, portability cues, and security-sensitive patterns.

Efficiency model

Token waste becomes visible fast.

Illustrative estimates only. Exact token and cost savings depend on the runtime, model, prompt framing, and skill quality.

Before optimization

~12,400 tokens per run. Loose trigger text, vague instructions, and noisy context increase token drag per activation.

After optimization

~4,100 tokens per run. Clear triggers, tighter IO, and portable helper scripts reduce repeated context and wasted turns.

How it works

Three steps from upload to structured feedback.

Upload a skill package, run deterministic checks, and review the structured result.

01. Upload SKILL.md plus supporting files

Provide a local folder or GitHub reference so the evaluator can inspect the full skill package instead of a disconnected markdown file.

02. Run deterministic checks

The package is validated and evaluated by the production evaluator. Optional AI review runs only after that deterministic step.

03. Get structured feedback

Results include score, quality tier, deterministic checklist, and optional review fields that are easy to scan and compare.

Why it matters

Skill quality affects cost, trigger behavior, and risk.

Well-written skills reduce wasted context, tighten activation boundaries, and expose security problems earlier.

Illustrative trigger quality

Baseline 72%, refined 94%. Agent runtimes often use a skill's name and description to decide when it should trigger.

Security signal

36.82% of public skills in Snyk's February 2026 ToxicSkills study had at least one security issue, and 13.4% had at least one critical issue.
